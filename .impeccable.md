## Design Context

### Users

個人網站定位為**個人品牌展示場**，不是業務導向的 landing page。訪客組合：
- 品牌端業主（marketing manager / 廣告主）評估是否接案
- Agency / 代理商同行看執行能力
- 設計/活動圈同業注意個人作品

**Job to be done**：快速讓訪客意識到「這是一個有強烈個人 POV 的 event producer，不是制式接案公司」，然後被其中一個案例或細節勾住、記住這個人。

**使用情境**：桌機瀏覽為主（業主上班時間逛），手機次之。光線環境多樣（辦公室白光、夜晚暗光）。注意力稀缺，首屏 3 秒內要建立辨識。

### Brand Personality

**三個詞：前衛・技術感・硬派。**

- 前衛：不走安全牌，敢用 HUD、glitch、TRANSMISSION LOG 這種不常見於活動產業的語言
- 技術感：像看情報簡報或觀測儀表，細節精準、資訊密度高
- 硬派：冷色調 + flame 作為唯一情緒色，拒絕可愛、溫馨、熱鬧、豐盛感

**情緒目標**：讓訪客感覺「這個人做事有紀律、有自己的 POV、不像其他活動公司」。不是要被喜歡，是要被記得。

### Aesthetic Direction

**已定調的風格語言**（v8 cold editorial，不是起點是現狀）：

- **調色盤**：ink navy `#0B1026` / paper `#FAFAF8` / flame `#E63E1F` / ice cyan `#5DD3E3`，四色鎖死 + 各自 tonal subscale
- **字體**：Fraunces（display/heading）+ Geist（body）+ Departure Mono（mono/label）+ Noto Serif/Sans TC + Chiron Hei/Sung HK WS（CJK 漸進增強）
- **細節語言**：HudCorners、GlitchText、FIELD KIT 分區、TRANSMISSION LOG blog、OSER 案例結構、`[ ]` 括號、serial number 序號、pixel-font 細節
- **首頁**：dark ambient（HomeAmbientBg shader），各 section `bg-transparent` 讓 shader 穿透
- **內頁**：paper 為主，editorial 排版
- **中英雙語**：next-intl，兩套文案都要寫得認真，不是 Google 翻譯

**Anti-references（明確排除）**：
- 一般台灣活動公司官網風（紅圓角按鈕、客戶 logo 牆、服務條列、首頁 banner 輪播）
- AI 味個人網站（紫漸層、Inter、icon 圓角、nested cards、hero metric 模板）
- Webflow/Framer 熱門設計師 portfolio 套版
- 但**可以**擁抱技術/dev 美學（HUD、terminal 字、glitch），這是刻意風格不是包袱

**Memory hook（訪客離開後該記得的）**：HUD/TRANSMISSION 這種**敘事質感**——對區塊命名、微動效、結構的處理，讓人覺得「這個人做東西有自己的語彙」。不是一張厲害的照片，而是整體語氣。

### Design Principles

1. **冷為底，flame 是例外**。Accent 是稀有的、會被記住的，不是到處撒。60-30-10 rule 裡 flame 屬於 10% 甚至更少。
2. **資訊密度優先於留白美**。HUD 語言本質是 dense readout，適度的密度比 Apple 式大留白更符合此網站的性格。
3. **細節命名即設計**。FIELD KIT / TRANSMISSION LOG / OSER 不是裝飾詞，是系統的一部分；新增區塊時要問「這一塊在 HUD 世界觀裡叫什麼？」
4. **雙語不是翻譯，是雙份創作**。EN 和 TC 要各自通順、各自符合該語言的節奏。TC 要考慮 CJK 字體的視覺重量。
5. **破格有目的**。GlitchText、asymmetric layout、serial number 序號——每一處破格都要能對應到「這是 intel report 的哪個慣例」，不是隨機。
6. **拒絕模板化**。如果某個 section 長得像其他活動公司官網 / Webflow template / AI 常見輸出，重做。
